## 3. Modal GLM-5.1 No Output

**Symptom:**  
- `opencode run "echo test" --model modal/glm-5.1-fp8` returned only `step_start`/`step_finish` with zero tokens and no assistant text.
- Direct `curl` to Modal API with leased key worked and returned `"content":"test"`.

**Root Causes Investigated:**
- **Proxy streaming behavior:** The original modal proxy on OCI VM (`/opt/modal-token-pool/modal_proxy.py`) returned SSE stream when `stream:true` or when path contained "stream". OpenCode expected a non-streaming response, causing an `IncompleteRead` or empty result.
- **Model name mismatch:** OpenCode model ID `modal/glm-5.1-fp8` vs Modal's actual model `zai-org/GLM-5.1-FP8`. The proxy did not translate the model name, leading to `404 Unknown model` for some request patterns.
- **Proxy version drift:** The OCI VM was running an older, unpatched version of the proxy, despite local changes. File copy via `scp`/`ssh` had not overwritten the system file due to permission/session issues.

**Fixes Applied:**
1. **Updated OCI VM proxy** with three critical changes:
   - **Force non-streaming:** For all requests, the proxy now rewrites the JSON body to set `"stream": false` before forwarding to Modal. This ensures compatibility with OpenCode's expected response format.
   - **Model ID mapping:** When `model` field equals `"glm-5.1-fp8"`, it is replaced with `"zai-org/GLM-5.1-FP8"` so Modal recognizes the model.
   - **Robustness:** Does not crash if `usage` is `null` in streaming chunks (by avoiding `.pop()` on `None`).

2. **Deployed fixed proxy:**
   - Wrote updated `modal_proxy.py` to `/opt/modal-token-pool/modal_proxy.py` on OCI VM via `scp` and `sudo mv`.
   - Restarted `modal-proxy.service` (systemd) successfully.

3. **OpenCode config:**
   - Added `"permission": "allow"` globally to stop all manual permission prompts.
   - Ensured Modal provider uses proxy baseURL: `http://92.5.60.87:4100/modal/v1` and API key `sk-sin-fleet-master`.
   - Model entry: `modal/glm-5.1-fp8`.
   - Removed experimental `useExtendedResponse` config that was not needed.

**Verification:**
- `curl -X POST http://92.5.60.87:4100/modal/v1/chat/completions -d '{"model":"glm-5.1-fp8",...}'` now returns normal JSON with `"content":"test"`.
- **OpenCode integration still pending final verification** due to possible daemon caching. User should restart OpenCode completely (close terminal/TUI and start fresh) and test:
  ```
  opencode run "echo test" --model modal/glm-5.1-fp8 --format json
  ```
  Expected: The `"part"` with `type: "text"` containing `test`.

## Important Notes for Future Maintenance

- **Modal model naming:**  
  Inside OpenCode config always use `modal/glm-5.1-fp8`. The proxy translates it to `zai-org/GLM-5.1-FP8`. Never change the provider's `baseURL` to point directly to Modal; always go through the pool proxy on `92.5.60.87:4100`.

- **Proxy streaming vs non-streaming:**  
  The proxy now **forces** `stream=false` for every request to maintain OpenCode compatibility. Do not remove this behavior; otherwise, OpenCode will hang or return empty results.

- **Qwen OAuth:**  
  The first login requires the browser step. After the user approves the device code once, the plugin stores the refresh token and rotates automatically. No further manual browser steps needed unless tokens expire.

- **Skills path:**  
  The `skills` key must be an object with `paths` array. Use the schema from this document.
